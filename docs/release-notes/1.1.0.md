# Systemd Gui 1.1.0

Systemd Gui 1.1.0 adds Quick Shell, improves service override handling and
polishes the update and documentation experience.

## Quick Shell

Quick Shell is a local `qs` command menu for your server shell. You manage the
menu in the Systemd Gui web interface, then run selected commands from SSH with
`qs`. It is meant for frequently used commands that you do not want to type or
remember every time.

- Create commands, nested categories and multi-line command sequences.
- Run direct paths such as `qs 1-2-3` or `qs 1 2 3`.
- Use placeholders such as `apt search {package}`; `qs` asks for the value
  before running the command.
- Print or copy commands instead of running them with `pN` / `printN` and
  `cN` / `copyN`.
- Optional shell integration supports shell-changing commands such as `cd /opt`.
- The shell history view can show recent commands, search them, run them again,
  and optionally show timestamps when the shell provides them.
- Import, export, back up and restore Quick Shell command sets, including a
  preview that shows what will be merged, replaced, skipped or renamed.

## Service Overrides

- Added a safer drop-in override editor for services.
- Shows when a service has active override files in the service list and detail
  view.
- Adds a flattened preview that folds override values into the unit file, making
  it easier to understand the effective configuration.
- Highlights added or changed override values in the merged preview.
- Keeps daemon-reload and restart prompts visible until the required action is
  handled or dismissed.

## Backups And Updates

- App update backups now include Quick Shell data together with favorites, notes,
  unit backups and environment-file backups.
- Quick Shell has its own server backups for command menus.
- Update and backup pages were reorganized into clearer Settings tabs.

## Documentation

- README now includes grouped feature sections for Systemd services, overrides,
  logs, Quick Shell, settings, security and updates.
- Added demo screenshots for the services overview, service detail page, Quick
  Shell, settings and the terminal `qs` menu.

## Notes

- This release keeps the public port default at `8850` and the internal Gunicorn
  bind default at `127.0.0.1:8851`.
- Existing installations can update through the Settings update page or by using
  the project's normal Git update path.
