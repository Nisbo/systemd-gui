Import / Export

Back up your Quick Shell categories or share selected commands with another Systemd Gui server.

Download full export
Import entries Import a full backup or a shared export file. You decide which category receives the entries and how name conflicts are handled.
Merge imported entries into selected category Leaves the selected category untouched and adds everything from the file inside it. If the file contains a category, that category is added as a child.
Replace entries inside selected category Keeps the selected category itself, deletes its current entries, then imports the file content into it. If the file contains a category, that category becomes a child entry.
Replace selected category Use this to update a command pack like APT. Select the existing category, then import a file that contains exactly one category. The selected category is replaced by that imported category.
Replace all Quick Shell categories Deletes all current Quick Shell categories and replaces them with the import file content. Settings from the file are used too.
Safe import Exact duplicates are skipped. If an imported entry has the same name but different content, it is renamed, for example APT (imported).
Skip identical only Exact duplicates are skipped, but entries with the same name and different content are still imported. You may get duplicate-looking names.
Import everything No duplicate checks are made. Everything from the file is imported as-is. If an entry or category with the exact same name already exists, you will get duplicate names in the selected category.
Duplicate handling not used This import mode replaces existing content first, so existing entries are not compared.
Import preview Select a JSON file to preview the import.

Shared commands can use placeholders such as apt search {package}. When selected in qs, the shell asks for the value first.
